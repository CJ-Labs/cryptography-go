// 2，开始根据s进行推算
// s*k = hash + r*私钥
// k = hash/s + r*私钥/s
// 引入 w = 1/s：
// k = hash/s + r*私钥/s
// k = hash*(1/s) + r*(1/s)*私钥
// k = hash*w + r*w*私钥
// 对比我们的目标形式：
// k = u1 + u2*私钥
// // 与
// k = hash*w + r*w*私钥
// // 对比可以看出：
// u1 必须等于 hash*w
// u2 必须等于 r*w

// ============================
// 3，当我们需要获得 u1 u2
// 我们想要把 k 表示成这种形式：
// k = u1 + u2*私钥
// 因为这样的话，两边乘以 G：
// k*G = (u1 + u2*私钥)*G
// k*G = u1*G + u2*私钥*G
// R = u1*G + u2*公钥      // 这样就可以用公钥验证了

// ================================
// 4，
// 计算第一个点 (point1)
// 	  把基点 G 重复加 u1 次
// 	  就像走路：向同一个方向走 u1 步
// 计算第二个点 (point2)
// 	  把小明的公钥重复加 u2 次
// 	  类似的，就像从公钥位置再走 u2 步
// 最后的验证
// 	  把这两个点加在一起，得到一个新的点 R'
// 	  如果这个新点的 x 坐标等于签名中的 r，说明签名是真实的
